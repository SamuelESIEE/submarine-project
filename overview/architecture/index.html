    System Architecture Overview

    This page provides a high-level view of the Autonomous Submarine system architecture, breaking down its main subsystems, their roles, and interfaces.

    1. System Overview

    The autonomous submarine integrates multiple subsystems to achieve reliable underwater navigation, sensing, communication, and power management. The architecture is designed for modularity, scalability, and fault tolerance.

    High-Level System Block Diagram

    2. Subsystems

    2.1 Core Processing Unit

    Handles navigation, sensor fusion, motor control, communication, and system management.
    - Main MCU: STM32H7 (high-performance ARM Cortex-M7)
    - Backup MCU: STM32L0 or ATTiny for low-power monitoring and wake-up functions

    Detailed MCU Firmware Design →

    2.2 Power Management

    Manages power sources, battery monitoring, and backup power to ensure continuous operation and safe shutdown.
    - Battery pack with voltage/current monitoring
    - Piezoelectric backup energy harvester and supercapacitor
    - Power regulation and switching circuits

    Detailed Power System Design →

    2.3 Sensors

    Provide environmental and navigation data, including:
    - Depth sensor (pressure-based)
    - Inertial Measurement Unit (IMU)
    - Temperature sensors
    - Battery voltage and current sensors

    Detailed Sensor System →

    2.4 Communication

    Handles data exchange with the surface and internal subsystems:
    - RF communication when surfaced
    - Internal buses: SPI, I2C, UART

    Detailed Communication Architecture →

    2.5 Actuators & Mobility

    Includes motors and motor drivers for underwater maneuvering:
    - 4 to 6 brushed/brushless DC motors
    - PWM speed control and safety cut-off mechanisms

    Detailed Actuator Design →

    3. Data Flow & Control Flow

    • Sensor data acquisition → MCU processing → Navigation decisions → Motor control commands
    • Logging sensor and status data to microSD card
    • Communication uplink/downlink when surfaced

    4. Safety & Fault Tolerance

    • Backup MCU monitors critical sensors and triggers emergency surface mode
    • Watchdog resets and external kill switch for fail-safe operation
    • Power fail detection and graceful shutdown
